Provider Adapter Pattern

Each provider implements one interface
interface
UsageProvider {
  name: string;
  fetchUsage(startDate, endDate): Promise<UsageRecord[]>;
  isConfigured(): boolean;
}

// Adapters:
AnthropicProvider → calls /v1/usage
GoogleAIProvider → calls Cloud Billing API
OpenAIProvider → calls /v1/organization/usage
KimiProvider → calls Kimi usage API

// Adding a new provider = one file implementing the interface

Key Decisions Summary

Stack

Next.js + SQLite + Drizzle ORM

Providers (v1)

Anthropic, Google AI, OpenAI, Kimi

Data Collection

Hourly cron polling, stored in SQLite

Dashboard

Overview (A) + Provider Cards (B) + Table (C)

Config

.env file — API keys + webhook URL + thresholds

Alerts

Configurable webhook (Slack default) on spend thresholds

Vercel (one-click deploy button) + delvestats.com
